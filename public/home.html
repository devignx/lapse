connect anywhere

Claude — web, mobile, desktopOAuth · no token

Settings → Connectors → Add custom connector → paste …/mcp. Leave client id & secret empty — registration is automatic. Log in with your Lapse account when redirected.

Claude Codebearer token

One command: claude mcp add --transport http lapse <url> --header "Authorization: Bearer <token>"

opencodebearer token

Five lines in ~/.config/opencode/opencode.json — the exact snippet is in Connections after you sign up.

Anything elseOAuth or token

Standard MCP over HTTP. OAuth-capable clients discover it from the URL alone; others send a bearer header. Local LLMs work too — LM Studio, Goose, any MCP client.

good to know

  • Sign up before authorizing. The Claude login page checks your Lapse account.
  • Tokens are shown once. They're stored hashed. Lost one? Rotate it in Connections — old token dies instantly, Claude's OAuth link is unaffected.
  • Login is rate-limited. A few wrong passwords per minute — wait 60s if you hit the wall.
  • Your entries are yours. Scoped to your account, invisible to other users. Passwords and tokens never stored in plaintext.
  • Deleting asks first. Agents are instructed to confirm before any delete.
